# M0: Planning And Architecture Baseline

## Goal

Create enough planning structure to begin implementation without turning the project into an unfocused IDE.

## Scope

- Document product vision and non-goals.
- Record key architecture decisions.
- Define source tree and CMake organization.
- Define milestone roadmap.
- Define engineering specs for agents, terminal, Git, GitHub, MCP, and storage.
- Define design specs for layout and interactions.

## Exit Criteria

- `docs/` contains plans, ADRs, milestone specs, engineering specs, and design specs.
- MVP scope is clear.
- First 20 implementation tasks are ordered.
- Major risks are documented with mitigations.

## Out Of Scope

- Application code.
- UI mock implementation.
- Build system implementation.
