# QTCode Project Planning Prompt

You are a senior software architect and KDE/Linux desktop application developer.

I want to design and plan a new desktop application called **QTCode**.

## Project Vision

QTCode is a lightweight native KDE/Linux developer cockpit focused on AI-assisted development, terminal workflows, GitHub integration, and project context management.

QTCode is **not** intended to become another VS Code, JetBrains IDE, or Electron application.

The goal is to create a fast, native Qt application for developers who primarily use AI agents and terminals rather than traditional code editors.

I spend most of my time vibe coding and rarely need a full editor.

QTCode should become the central workspace for:

* AI coding agents
* GitHub repositories
* Terminal workflows
* Project memory and RAG systems
* Build/test execution
* Git operations
* GitHub Issues and Pull Requests

The editor should be optional and secondary.

# AI Agent Panel Requirements

The AI panel is a primary feature.

The user should be able to:

* Select an active AI agent
* Switch agents per project
* Open multiple conversations
* Save sessions
* View retrieved context
* View retrieved project memory
* View diffs generated by agents
* Approve or reject changes

Supported agents:

* Codex
* Cursor
* GitHub Copilot
* Claude
* aider

Future agents should be easy to add.

# Optional Editor Features

The editor is not the product center, but QTCode now includes focused workspace file editing through KTextEditor (see ADR 0011).

If included beyond that slice:

* KTextEditor workspace tabs for small edits
* Diff viewer
* Read-only code browsing for review

Avoid building a full IDE. See [KTextEditor workspace spec](docs/specs/ktexteditor-workspace-spec.md) for non-goals.
